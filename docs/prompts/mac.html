Homebrew 설치

Homebrew는 macOS용 패키지 관리자입니다. "앱스토어 없이 터미널로 프로그램 설치"라고 생각하면 됩니다. 터미널(Terminal.app 또는 iTerm2)을 열고 아래 명령을 붙여넣으세요.

bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
💡
팁: Apple Silicon(M1/M2/M3/M4) Mac이면 설치 후 아래 명령도 실행하세요. Homebrew 경로를 쉘에 등록합니다.
bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

시장 체제 5개 센서 이해하기

시장이 "좋은 상태"인지 "위험한 상태"인지 판단하는 5개 신호입니다. 각 신호에 가중치를 곱해 점수를 계산하고, 점수에 따라 투자 전략을 바꿉니다.

센서가중치쉬운 설명데이터 소스
VIX30%주식 시장의 "공포 온도계". 20 아래면 안전, 30 이상이면 패닉 상태^VIX
Trend25%SPY(S&P500 ETF)가 50일/200일 이동평균선 위에 있으면 "상승 추세"SPY vs SMA
Breadth18%RSP(동일가중 ETF) vs SPY — 소형주도 오르면 시장 전체가 건강한 것RSP/SPY 비율
Credit15%HYG(회사채)/IEF(국채) 비율 — 투자자가 위험 자산을 기피하면 이 비율이 낮아짐HYG/IEF
Yield Curve12%10년 금리 - 3개월 금리. 마이너스(장단기 역전)면 경기 침체 신호^TNX - ^IRX
점수 → 체제 판단: 0.00 ~ 0.75 → risk_on (공격 투자 가능) 0.75 ~ 1.50 → neutral (중립) 1.50 ~ 2.25 → risk_off (방어 투자) 2.25 이상 → crisis (현금 보유)

1

GBM이 뭔가요? (초보자 설명)

GBM(Gradient Boosting Machine)은 "약한 예측기 여러 개를 합쳐서 강한 예측기를 만드는" 알고리즘입니다.

💡
비유: Excel의 예측 함수(FORECAST)가 1명의 예측이라면, GBM은 100명의 예측을 평균 낸 결과입니다. 훨씬 정확합니다. 주식 방향 예측, 신용카드 사기 탐지, 날씨 예보 등에 실제로 쓰입니다.
2

Walk-Forward 검증이란?

일반 방식 (잘못된 방법): 전체 데이터 → 학습 + 테스트 (미래 데이터로 과거 테스트 → 사기!) Walk-Forward (올바른 방법): 1월~6월 학습 → 7월 예측 → 성능 측정 1월~7월 학습 → 8월 예측 → 성능 측정 1월~8월 학습 → 9월 예측 → 성능 측정 ... 실제 투자 환경과 동일한 방식으로 검증!
📌
참고: Walk-Forward 검증을 통과한 모델만 실제 예측에 사용합니다. 이 검증이 없으면 "백테스트에서만 잘 되는 가짜 전략"이 나올 수 있습니다.

Phase 8

자동화 설정 (launchd)

⏱ 15분
🎯 이 단계의 목표: 매일 오전 7시에 분석이 자동으로 실행되도록 macOS launchd 스케줄러를 설정합니다.
1

launchd란?

📌
참고: launchd는 macOS의 작업 스케줄러입니다. Windows의 "작업 스케줄러"와 같은 역할을 합니다. 컴퓨터가 켜져 있으면 지정한 시간에 자동으로 스크립트를 실행합니다.
2

자동 실행 등록

bash
cd ~/workspace/us-stock
source .venv/bin/activate

# 매일 오전 7시 자동 실행 등록
python scripts/run_daily_scheduler.py --install-cron --time 07:00

# 상태 확인
python scripts/run_daily_scheduler.py --status

# 즉시 1회 실행 테스트
python scripts/run_daily_scheduler.py
⚠️
주의: 자동 실행 시 .env 파일의 API 키가 올바르지 않으면 에러가 발생합니다. 로그는 logs/daily_run_YYYYMMDD.log에서 확인하세요.
3

launchd plist 파일 직접 수정 (고급)

bash
# 실행 시간 변경 (예: 오전 6시로 변경)
# ~/Library/LaunchAgents/com.us-stock.daily.plist 편집 후:
launchctl unload ~/Library/LaunchAgents/com.us-stock.daily.plist
launchctl load ~/Library/LaunchAgents/com.us-stock.daily.plist

# 로그 확인
tail -50 ~/workspace/us-stock/logs/daily_run_$(date +%Y%m%d).log
