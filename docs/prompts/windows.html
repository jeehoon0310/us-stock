winget으로 Python 3.13 + Node.js 20 설치

winget은 Windows 11/10에 기본 포함된 패키지 관리자입니다. PowerShell에서 실행하세요.

powershell
# Python 3.13 설치
winget install Python.Python.3.13

# Node.js 20 LTS 설치
winget install OpenJS.NodeJS.LTS

# 설치 후 PowerShell 재시작, 그 다음 버전 확인
python --version    # Python 3.13.x 출력 확인
node --version      # v20.x.x 출력 확인
npm --version       # 10.x.x 출력 확인
🚫
Windows Defender 차단 대처: Python 설치 시 "Windows Defender SmartScreen" 경고가 나오면 "추가 정보" → "실행"을 클릭하세요. 공식 Microsoft Store 경로를 통한 설치는 안전합니다.
💡
winget이 없으면? python.org와 nodejs.org에서 직접 인스톨러를 다운로드하세요. Python 설치 시 반드시 "Add Python to PATH" 체크박스를 선택하세요.

(선택 권장) WSL2 설치

WSL2(Windows Subsystem for Linux)를 설치하면 리눅스 명령어를 Windows에서 그대로 쓸 수 있습니다. yfinance, curl_cffi 같은 라이브러리가 WSL2에서 더 안정적으로 동작합니다.

powershell
# 관리자 PowerShell에서 실행
wsl --install

# 재부팅 후 Ubuntu 설정 완료
# Ubuntu 터미널에서는 macOS 가이드의 bash 명령을 그대로 사용 가능
📌
참고: WSL2를 사용하면 이 가이드의 나머지 단계 대신 macOS 가이드의 명령어를 그대로 사용할 수 있습니다. Windows 네이티브로 진행하려면 이 가이드를 계속 따르세요.

시장 체제 5개 센서 이해하기

Trend25%SPY(S&P500 ETF)가 50일/200일 이동평균선 위에 있으면 "상승 추세"SPY vs SMA

Phase 8

자동화 설정 (작업 스케줄러)

⏱ 15분
🎯 이 단계의 목표: 매일 오전 7시에 분석이 자동으로 실행되도록 Windows 작업 스케줄러를 설정합니다.
1

PowerShell로 작업 스케줄러 등록 Windows 방식

macOS launchd 대신 Windows Task Scheduler를 사용합니다. 관리자 권한 PowerShell에서 실행하세요.

powershell
# 작업 스케줄러 등록 스크립트 생성
$scriptPath = "C:\workspace\us-stock\scripts\run_scheduler.bat"
@"
@echo off
cd /d C:\workspace\us-stock
call .venv\Scripts\activate.bat
python scripts\run_daily_scheduler.py >> logs\scheduler.log 2>&1
"@ | Set-Content $scriptPath

# 매일 오전 7시 자동 실행 등록
$action = New-ScheduledTaskAction -Execute $scriptPath
$trigger = New-ScheduledTaskTrigger -Daily -At "07:00"
$settings = New-ScheduledTaskSettingsSet -ExecutionTimeLimit (New-TimeSpan -Hours 2)
Register-ScheduledTask -TaskName "USStockAnalysis" -Action $action -Trigger $trigger -Settings $settings -RunLevel Highest

Write-Host "작업 스케줄러 등록 완료!" -ForegroundColor Green
⚠️
주의: 자동 실행 시 .env 파일의 API 키가 올바르지 않으면 에러가 발생합니다. 로그는 logs\scheduler.log에서 확인하세요.
2

작업 스케줄러 관리

powershell
# 등록 확인
Get-ScheduledTask -TaskName "USStockAnalysis"

# 즉시 실행 테스트
Start-ScheduledTask -TaskName "USStockAnalysis"

# 작업 삭제 (필요 시)
Unregister-ScheduledTask -TaskName "USStockAnalysis" -Confirm:$false

# 로그 확인
Get-Content C:\workspace\us-stock\logs\scheduler.log -Tail 50

❓ 자주 묻는 질문

작업이 실행됐는데 결과 파일이 없으면?
logs\scheduler.log를 확인하세요. .venv 경로나 Python 경로가 잘못됐을 가능성이 높습니다. bat 파일을 직접 더블클릭해서 실행해보세요.
컴퓨터가 꺼져 있으면 자동 실행 안 되나요?
Windows 작업 스케줄러는 컴퓨터가 켜져 있을 때만 실행됩니다. "조건" 탭에서 "컴퓨터가 절전 모드일 때 절전 모드 해제" 옵션을 활성화하면 절전 상태에서 자동으로 깨어납니다.
