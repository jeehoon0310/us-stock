## 후보 A: Idiosyncratic Volatility (IVOL) — 7번째 스크리닝 팩터

**카테고리**: 신규 팩터 (Risk anomaly)
**근거**:
- [Ang, Hodrick, Xing, Zhang — "High Idiosyncratic Volatility and Low Returns" (Columbia)](https://business.columbia.edu/sites/default/files-efs/pubfiles/3361/ang_high_idiosyncratic_volatility.pdf)
- [The alphas of beta and idiosyncratic volatility — ScienceDirect 2022](https://www.sciencedirect.com/science/article/abs/pii/S1386418122000131)
- [Alpha Architect — Idiosyncratic Volatility: what role does it play in returns?](https://alphaarchitect.com/idiosyncratic-volatility/)

**핵심 아이디어**: 시장(SPY)으로 설명되지 않는 잔차 변동성이 높은 종목은 평균적으로 **낮은 forward return**. CAPM 회귀 잔차의 21일 표준편차로 측정. 기존 Volume(거래량 비정상)·Technical(추세) 팩터와 개념적으로 직교. Fama-French 3-factor alpha 통계적 유의성이 50년 입증된 anomaly.

**구현 방법**:
```python
# src/analyzers/idio_vol_factor.py
import numpy as np
import statsmodels.api as sm

def compute_ivol(stock_returns: pd.Series, spy_returns: pd.Series, window: int = 21) -> pd.Series:
    """21일 롤링 CAPM 잔차의 표준편차 (T-1까지만 사용)."""
    aligned = pd.concat([stock_returns, spy_returns], axis=1).dropna()
    aligned.columns = ["r_i", "r_m"]
    ivol = pd.Series(index=aligned.index, dtype=float)
    for end in range(window, len(aligned)):
        win = aligned.iloc[end - window : end]  # T-1까지만
        X = sm.add_constant(win["r_m"])
        resid = sm.OLS(win["r_i"], X).fit().resid
        ivol.iloc[end] = resid.std() * np.sqrt(252)
    return ivol

# 점수: 낮은 IVOL → 높은 점수 (cross-sectional rank 역순)
score = 100 * (1 - ivol_today.rank(pct=True))
```

**예상 임팩트**: 보수적 추정 IC +0.005~0.012 (논문 0.02 주장의 30%), Sharpe +0.04~0.08

## 후보 B: CBOE Equity Put/Call Ratio — 6번째 체제 센서

**카테고리**: 신규 체제 센서 (Sentiment)
**근거**:

**핵심 아이디어**: 인용 — "장기 풋 누적이 강세장에서 발생하면 기관 디리스킹 신호이며 체제 전환의 가장 신뢰할 만한 leading indicator". CBOE Equity PCR은 ^CPCE 티커로 yfinance에서 무료. 5일 EMA가 1.0 초과 + 20일 z-score > 1.5면 위험 회피 체제로 가중. 기존 5센서가 모두 "표면 가격"인 반면 PCR은 "옵션 시장의 비대칭 헤지 비용" — 직교 정보원.

**구현 방법**:
```python
# src/analyzers/market_regime.py — _compute_pcr_signal 추가
def _compute_pcr_signal(self) -> tuple[float, dict]:
    """CBOE Equity PCR의 z-score 기반 체제 신호 (T-1)."""
    pcr = yf.download("^CPCE", period="6mo", session=self.session)["Close"]
    pcr = pcr.iloc[:-1]  # T-1만 (look-ahead 방지)
    pcr_ema5 = pcr.ewm(span=5).mean()
    z20 = (pcr_ema5.iloc[-1] - pcr_ema5.tail(20).mean()) / pcr_ema5.tail(20).std()

    # 0(risk_on) ~ 3(crisis) 스케일로 매핑
    if z20 > 1.5:   score = 2.5    # 풋 폭증 = 디리스킹
    elif z20 > 0.5: score = 1.5
    elif z20 < -1:  score = 0.3    # 콜 과열 = euphoria
    else:           score = 1.0
    return score, {"pcr_ema5": float(pcr_ema5.iloc[-1]), "z20": float(z20)}
```

**적용 위치**: `src/analyzers/market_regime.py` — `MarketRegimeDetector` 클래스에 6번째 센서 추가. 가중치 재배분: VIX 30→27, Trend 25→23, Breadth 18→16, Credit 15→14, YieldCurve 12→10, **PCR 10**
**예상 임팩트**: 체제 전환 감지 정확도 +3~7%p (특히 강세장 후반 risk_off 조기 감지). 2018-12, 2022-01 같은 변곡점에서 lead 5~15 거래일
**구현 난이도**: 낮음 (≤2h) — yfinance ^CPCE 단일 티커
**데이터 요구**: 기존 yfinance 충분 (^CPCE 검증 필요 — 실패 시 CBOE 공식 CSV fallback)

## 후보 C: Rolling IC 기반 Alpha Decay Monitor — 메타 가중치 조정

**카테고리**: ML 기법 / 메타 레이어
**근거**:
- [Not All Factors Crowd Equally: Modeling, Measuring, and Trading on Alpha Decay (arXiv 2512.11913)](https://arxiv.org/pdf/2512.11913)
- [VertoxQuant — Strategy Decay Detection: Building a Warning System for Alpha Erosion](https://www.vertoxquant.com/p/strategy-decay-detection)
- [PyQuantNews — Real Factor Alpha with Information Coefficient and Alphalens](https://www.pyquantnews.com/free-python-resources/real-factor-alpha-how-to-measure-it-with-information-coefficient-and-alphalens-in-python)

**핵심 아이디어**: 6개 팩터별 **63일 rolling Spearman IC**를 추적하고, IC가 음수이거나 t-stat < 1로 떨어진 팩터는 가중치를 자동으로 50%로 감쇄. 2025년 12월 논문은 "기계적 팩터의 디케이는 hyperbolic α(t) = K/(1+λt)"로 모델링되며 선형/지수 모델보다 우월하다고 보고. 현재 시스템은 6팩터 가중치가 **고정** — 팩터 크라우딩 발생 시 그대로 손실 흡수.

**구현 방법**:
```python
# src/analyzers/alpha_decay_monitor.py
def compute_factor_ic(factor_scores: pd.DataFrame, fwd_returns: pd.DataFrame,
                      window: int = 63) -> pd.DataFrame:
    """팩터별 rolling IC (cross-sectional Spearman per day)."""
    daily_ic = factor_scores.corrwith(fwd_returns, axis=1, method="spearman")
    return daily_ic.rolling(window).mean()  # 63일 평활 IC

def adaptive_weights(base_weights: dict, rolling_ic: dict) -> dict:
    """IC 신호별 가중치 조정 — t-stat 기반."""
    adjusted = {}
    for factor, w in base_weights.items():
        ic_mean = rolling_ic[factor].iloc[-1]
        ic_std  = rolling_ic[factor].rolling(63).std().iloc[-1]
        t_stat  = ic_mean / (ic_std / np.sqrt(63)) if ic_std > 0 else 0
        decay   = 0.5 if t_stat < 1.0 else 1.0  # IC 약화 시 50% 감쇄
        adjusted[factor] = w * decay
    # 합 1로 재정규화
    s = sum(adjusted.values())
    return {k: v / s for k, v in adjusted.items()}
```

**적용 위치**: `src/analyzers/smart_money_screener_v2.py` — `_calculate_composite_score()` 호출 전에 `adaptive_weights()`로 가중치 동적 조정. 새 모듈 `src/analyzers/alpha_decay_monitor.py` 생성, `output/factor_ic_history.parquet` 누적 저장.
**예상 임팩트**: 팩터 크라우딩 시기(예: 2024 momentum crash) 손실 -30~50% 감쇄. Sharpe +0.05~0.10. 정상 시기에는 거의 영향 없음 (gating 효과)
**구현 난이도**: 중간 (≤4h) — fwd_return 라벨링 + IC 누적 인프라 필요
**데이터 요구**: 기존 OHLCV 충분 (forward 21일 return 자체 계산)
**리스크**: 63일 IC는 노이즈 큼 — t-stat threshold 1.0이 너무 민감하면 정상 팩터도 감쇄. 백테스트로 threshold 0.8~1.5 sweep 필요. Look-ahead 방지: IC 계산 시 반드시 fwd_return은 T+21까지 실현된 데이터만
