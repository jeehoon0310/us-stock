> [!summary]
> 한국투자증권 Open API (open-trading-api + kis-ai-extensions)를 활용한 자동 매수/매도 전략 설계 가이드. Claude Code MCP 연동으로 자연어 기반 전략 설계, 백테스트, 모의투자 실행까지 풀 파이프라인을 구현한다.

## 1. KIS Open API 생태계

### open-trading-api
- GitHub: https://github.com/koreainvestment/open-trading-api
- Stars: 1,200+ / Forks: 646+
- Language: Python 90.1%, TypeScript 7.3%
- 구조: examples_llm/(LLM 에이전트용 모듈화 예제), examples_user/(실 사용자용 통합 코드), strategy_builder/(전략 UI), backtester/(QuantConnect Lean), MCP/(AI 도구 통합)
- 지원 상품: 국내주식, 해외주식, 국내채권, 국내선물옵션, 해외선물옵션, ETF/ETN, ELW
- 기능: REST API(주문/조회) + WebSocket(실시간 호가/체결/통보)

### kis-ai-extensions
- GitHub: https://github.com/koreainvestment/kis-ai-extensions
- Stars: 121 / NPM: @koreainvestment/kis-quant-plugin
- Claude Code, Cursor, Codex, Gemini CLI 전용
- Skills: kis-strategy-builder, kis-backtester, kis-order-executor, kis-team, kis-cs
- Safety Hooks: kis-prod-guard, kis-secret-guard, kis-trade-log, kis-mcp-log

## 2. 설치 및 설정

### 사전 요구사항
- Python 3.11+, Node.js 18+, Docker Desktop (백테스터용)
- KIS 계정 + Open API 신청 승인 (https://apiportal.koreainvestment.com/)

### 설치
```bash
# 메인 리포지토리
git clone https://github.com/koreainvestment/open-trading-api.git
cd open-trading-api
uv sync

# Claude Code 확장
npx @koreainvestment/kis-quant-plugin init --agent claude
```

### 설정 파일
```yaml
# ~/.KIS/config/kis_devlp.yaml
my_app: "your_real_app_key"
my_sec: "your_real_app_secret"
my_acct: "12345678"
my_acct_prdt: "01"
paper_app: "your_paper_app_key"
paper_sec: "your_paper_app_secret"
my_paper_stock: "87654321"
```

### Claude Code 슬래시 커맨드
- `/auth vps` — 모의투자 인증
- `/auth prod` — 실전 인증
- `/my-status` — 보유종목, 잔고, 지수 조회
- `/kis-setup` — 환경 자동진단
- `/kis-help [에러코드]` — KIS 에러코드 설명

## 3. 10대 프리셋 전략

| # | 전략명 | 유형 | 설명 | 매수조건 | 매도조건 | 적합한 시장 |
|---|--------|------|------|----------|----------|-------------|
| 01 | 골든크로스 | Trend Following | 단기MA와 장기MA 교차 신호 활용 | SMA(5/20) > SMA(60/120) | 이동평균 역전 시 | 상승장 |
| 02 | 모멘텀 | Trend Following | N일 수익률 최고 종목 선별 | N일 수익률 상위 종목 진입 | 순위 이탈 또는 손절 | 강한 추세장 |
| 03 | 52주 신고가 | Breakout | 가격이 52주 최고가를 돌파하는 순간 진입 | 52주 최고가 돌파 | 고점 대비 -N% 하락 | 강한 상승장 |
| 04 | 연속 상승/하락 | Trend Following | N일 연속 방향성 감지 후 추세 추종 | N일 연속 상승 확인 | N일 연속 하락 전환 | 모멘텀 장 |
| 05 | 이격도 | Mean Reversion | 가격과 이동평균의 괴리율로 과열/침체 판단 | 가격/MA 비율 < 0.95 | 가격/MA 비율 > 1.05 | 박스권 |
| 06 | 돌파 실패 | Stop Loss | 고점 돌파 후 반전 시 즉시 손절 | 다른 전략과 조합 사용 | 돌파 후 반전 감지 즉시 | 모든 시장 |
| 07 | 강한 종가 | Momentum | 종가 위치 + 거래량으로 매수세 강도 측정 | 종가 > (고가+저가)/2 + 거래량 증가 | 약한 종가 + 거래량 감소 | 상승장 전환 |
| 08 | 변동성 확장 | Breakout | ATR 급등으로 추세 전환 초입 포착 | ATR 급등 + 방향성 확인 | ATR 수축 또는 반대 방향 | 횡보 후 추세 전환 |
| 09 | 평균회귀 | Mean Reversion | 볼린저밴드 이탈 후 평균 복귀 타이밍 포착 | 하단 이탈 후 밴드 내 복귀 | 상단 이탈 또는 중심선 도달 | 박스권/약세장 |
| 10 | 추세 필터 | Trend Following | 200MA 기준으로 전체 추세 방향 필터링 | 200MA 위에서만 매수 허용 | 200MA 아래로 이탈 시 | 중장기 상승장 |

## 4. 자동매매 구현 코드 예시

### 4-1. 전략 파일 (.kis.yaml)
```yaml
strategy_name: "Golden Cross"
indicators:
  - name: "SMA_20"
    type: "simple_moving_average"
    period: 20
  - name: "SMA_60"
    type: "simple_moving_average"
    period: 60
signals:
  buy:
    condition: "SMA_20 > SMA_60"
  sell:
    condition: "SMA_20 < SMA_60"
risk_management:
  stop_loss: -5%
  trailing_stop: -3%
  position_size: 10%
```

### 4-2. Python 인증 및 주문
```python
import kis_auth as ka

# 모의투자 인증
ka.auth(svr="vps", product="01")

# 보유종목 조회
from domestic_stock_functions import get_holdings
holdings = get_holdings(ka.get_token())

# 매수 주문
from domestic_stock_functions import execute_order
result = execute_order(
    token=ka.get_token(),
    stock_code="005930",  # 삼성전자
    qty=10,
    order_type="buy",
    price=0  # 시장가
)

# 매도 주문
result = execute_order(
    token=ka.get_token(),
    stock_code="005930",
    qty=10,
    order_type="sell",
    price=0
)
```

### 4-3. WebSocket 실시간 데이터
```python
import kis_auth as ka

ka.auth()
ka.auth_ws()

kws = ka.KISWebSocket(api_url="/tryitout")

# 실시간 호가 구독
kws.subscribe(
    request={"data": ["005930", "000660"]}  # 삼성, SK하이닉스
)

# 콜백에서 가격 변동 감지 → Stop Loss 실행
```

## 5. Claude Code 자동매매 데모 시나리오

### 시나리오 1: 전략 설계 → 백테스트 → 모의투자
```
사용자: "골든크로스 전략 만들어줘"
Claude Code → kis-strategy-builder 활성화
→ .kis.yaml 생성 (사용자 확인)

사용자: "삼성전자 1년 백테스트 해줘"
Claude Code → kis-backtester 활성화
→ Docker QuantConnect Lean 실행
→ HTML 성과 리포트 (Sharpe, MDD, 승률, 수익곡선)

사용자: "결과 괜찮으니 모의투자로 실행"
Claude Code → kis-order-executor 활성화
→ /auth vps (모의투자 인증)
→ 현재 시그널 확인 → 매수/매도 주문 실행
→ WebSocket으로 실시간 체결 모니터링
```

### 시나리오 2: us-stock 시그널 연동
```
사용자: "us-stock 분석 결과로 한투 모의투자 해줘"
Claude Code:
1. us-stock 파이프라인 실행 → Verdict: GO
2. Top Picks에서 Grade A 종목 필터
3. 한투 API로 해외주식 모의 매수 (position_size = 자본의 5%)
4. Trailing Stop -5% 설정
5. WebSocket으로 가격 모니터링
6. Stop Loss 트리거 시 자동 매도
```

### 시나리오 3: 전체 파이프라인 (kis-team)
```
사용자: "전략부터 주문까지 다 해줘"
Claude Code → /kis-team 활성화
Step 1: Strategy Builder → 전략 설계 (사용자 확인)
Step 2: Backtester → 성과 검증 (사용자 리뷰)
Step 3: Order Executor → 주문 실행 (사용자 최종 확인)
```

## 6. 안전장치 상세

| 안전장치 | 기능 | 트리거 조건 |
|----------|------|------------|
| kis-prod-guard | 실전 주문 시 사용자 확인 필수 | svr="prod" 환경에서 주문 실행 시 |
| kis-secret-guard | API 키 하드코딩 방지 | 코드에 key/secret 문자열 감지 시 |
| kis-trade-log | 모든 거래 감사 추적 | 매수/매도/취소 주문 실행 시 |
| kis-mcp-log | 백테스트/최적화 호출 추적 | MCP 도구 호출 시 |
| 모의/실전 분리 | 별도 계정번호, 별도 인증 | svr 파라미터로 완전 분리 |
| Rate Limit | API 호출 제한 보호 | 초당/분당 호출 한도 초과 시 |
