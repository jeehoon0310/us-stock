- Claude Code 멀티계정 아키텍처 drawio 시연 (80.References/82.Manuals/AI/ClaudeCode/drawio/Claude Code 멀티계정 아키텍처.drawio):
	- claude-a, claude-b, claude-c, claude-cloud
	- 4개 계정이 ~/.claude 마스터 설정을 symlink로 공유
	- 17개 MCP 서버 동시 연결 (Atlassian, Gmail, Slack, Notion, Obsidian, Google Drive 등)
	- 각 계정별 격리 (.claude.json만 분리)
- "4개 계정이 동시에 다른 일을 합니다. 하나가 회의록 정리하는 동안 다른 하나는 코드를 짜고, 또 다른 하나는 리서치를 합니다."
- **비용 최적화 3개 설정:**
	- Thinking 토큰 10,000 상한 -> 60% 절감 (불필요한 '생각' 제거)

### 2-2b. YouTube->Telegram->LLM Wiki 파이프라인 (3분)

**토킹 포인트:**
- 비전문가 설명: "유튜브에서 봐야 할 영상을 텔레그램(카톡 같은 앱)에 저장합니다. Claude Code가 자동으로 자막을 뽑고, 요약하고, 주제별로 정리합니다. 비서가 영상을 대신 보고 핵심만 메모해주는 것과 같습니다"
- LLM Wiki = "AI의 두뇌 노트"
  - 안드레 카파시(테슬라 자율주행 AI 총괄 출신)가 제안한 방식
  - 비유: "사람으로 치면 인수인계 문서입니다. AI는 대화가 끝나면 기억을 못 합니다. 매번 인수인계를 받아야 하는데, 3,551개 파일을 다 읽을 수 없으니 22KB(A4 5장)짜리 요약본을 만들어둡니다. 이 5장만 읽으면 3,500페이지를 다 읽은 효과"
  - 실제 수치: 22KB wiki = 3,551개 파일 요약, 토큰 85-95% 절감
- 워크플로우: Telegram Saved -> URL 추출 + SQLite 중복제거 -> NotebookLM 보고서 -> Obsidian 주제별 저장 -> Wiki 자동 반영
- "이렇게 모은 최신 트렌드가 us-stock 시스템에도 자동으로 반영됩니다"

**시각자료:**
- LLM Wiki v3 아키텍처 drawio 화면
- Telegram->YouTube 파이프라인 워크플로우 스크린샷

**감정:** 이해, 감탄

- "유튜브 112편 만들면서 콘텐츠 리서치도 자동"

**데모 4: 500종목 5분 분석**

- 여름(확장): 금리 상승, 수익 정점 -> 보유, 위험 줄이기
- 가을(둔화): 금리 높음, 수익 하락 -> 방어주, 채권
- 겨울(침체): 금리 하락, 수익 바닥 -> 축적
- 코스톨라니의 개 산책 비유 -- 단기 괴리는 크지만 장기적으로 수렴
- 채플린의 말 -- 나스닥 단기 하락은 비극, 장기 차트는 희극(우상향)
- 3개 숫자 치트시트:
	1. Yield Curve (FRED T10Y2Y) -- 경제 체온계
	2. High Yield Spread -- 부도 위험 미터
	3. Forward 12M EPS -- 시장의 미래 기대
- "이 3개 숫자가 시장 방향의 80%를 말해줍니다"
- 현재 상황: Fed 3.50-3.75%, Higher for Longer, Yield Curve 정상화 중
- 지금은 어떤 계절인지 판단하는 법

**시각자료:**
- 4계절 순환 다이어그램 (봄/여름/가을/겨울 + 각 전략)
- 3개 숫자 치트시트 대시보드
- FRED 차트 (T10Y2Y, HY Spread, Forward EPS)

### 4-3. 밸류에이션 기초 (7분)

> "주가 = EPS x PER" — John Burr Williams, Theory of Investment Value (1938)

> "측정되지 않는 것은 관리되지 않는다." — Drucker (통상 귀속)

**토킹 포인트:**
- 가격 = EPS x PER
  - EPS = 1주당 순이익 (회사가 얼마나 버는지)
  - PER = 시장이 얼마나 기대하는지 (비싸다/싸다)
- 윌리엄스가 1938년에 정의한 공식이 지금도 유효하다
- 드러커의 말처럼 측정해야 관리할 수 있다 -- 재무분석이 필요한 이유
- DuPont ROE = 이익률 x 자산회전 x 레버리지
  - "레버리지로 높은 ROE = 위험. 마진으로 높은 ROE = 품질"
  - 예: 부채로 ROE 30% vs 마진으로 ROE 30% -- 같은 숫자, 완전히 다른 의미
- Fed Model: 이익수익률(1/PER) vs 10년 국채 -- 높은 쪽이 승리
  - S&P 500 이익수익률 5.05% vs 국채 4.3% -> 주식이 약간 매력적
- S&P 500 Forward PER 19.8x (현재)
- EPS 매트릭스:

| | PER 상승 | PER 하락 |
|---|----------|----------|
| EPS 상승 | 좋음 | 최고 |
| EPS 하락 | 최악 | 나쁨 |

**시각자료:**
- 가격 = EPS x PER 공식 슬라이드
- DuPont 분해 다이어그램
- Fed Model 비교 차트
- EPS 매트릭스 2x2

### 4-4. 채권 기초 (3분)

> "Malkiel's 5 Theorems -- 채권 가격과 금리의 관계 5대 정리" — Burton Malkiel, A Random Walk Down Wall Street

> "워털루 전투에서 로스차일드가 가장 빨리 정보를 얻어 채권을 매수했다. 이것이 근대 채권 투자의 기원이다." — 역사적 일화

**토킹 포인트:**
- 채권 가격과 금리는 역의 관계 (시소)
- 말키엘 5법칙 요약:
	1. 가격과 수익률 반대 방향
	2. 만기 길수록 변동성 큼
	3. 만기 길수록 금리 하락 시 수익 큼
	4. 볼록성 존재 (상승 > 하락)
	5. 낮은 쿠폰 + 긴 만기 = 최대 볼록성
- Duration 개념: 금리 1% 변동 -> 채권 가격 이동 %
	- Duration 5 = 금리 1% 상승 시 채권 가격 약 5% 하락
- "채권을 사라는 게 아닙니다. 채권을 이해해야 주식을 이해합니다"
	- Yield Curve가 주식에 미치는 영향
	- 채권 시장은 주식 시장보다 똑똑하다

**시각자료:**
- 채권 가격/금리 시소 일러스트
- 말키엘 5법칙 요약 카드
- Duration 효과 그래프
